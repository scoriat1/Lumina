# AGENTS.md

For changes affecting login, clients, packages, scheduling, session editing, or related UI/API behavior, done means implementation plus appropriate validation for the area changed.

Modes:
- Normal local dev: use the normal local database and startup path by default.

Do not silently switch normal development to a separate test-only database.

Final report must include:
- files changed
- commands run
- pass/fail status
- assumptions or blockers
